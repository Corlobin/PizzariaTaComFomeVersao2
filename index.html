A pizzaria

A pizzaria “Tá com fome” é especialista em desenvolver pizzas em formato de cone e pizzas tradicionais. As pizzas podem possuir os seguintes ingredientes: massa pizza tradicional, massa pizza cone, molho de tomate, queijo parmesão, queijo mussarela, queijo margherita, queijo provolone, queijo gongorzola, cebola e presunto. Atualmente o cozinheiro prepara as seguintes pizzas:

  • a. Mussarela:
  • i. Ingredientes: massa, molho de tomate, queijo mussarela, queijo gongorzola, cebola
  • e presunto;
  • b. Quatro queijos:
  • i. Ingredientes: massa, molho de tomate, queijo parmesão, queijo mussarela, queijo margherita, queijo provolone, queijo gongorzola, cebola e presunto;

É importante ressaltar que a massa é modifica conforme o tipo da pizza, ou seja, caso a pizza seja tradicional é utilizado a massa pizza tradicional. Caso contrário, a massa escolhida é a massa pizza cone. Cada componente da pizza possui um valor distinto. Os queijos custam 1,00. O molho de tomate, a cebola e o presunto custam 3,00, cada um. A massa tradicional custa 5,00 e a massa de cone custa 10,00. A soma de cada componente é o valor da pizza. Cada classe possui um método preço. Para melhorar o desempenho da pizzaria será desenvolvido um software para gerenciar o preparo das pizzas. O projetista especificou as seguintes decisões de projeto:

  • Utilizar o padrão decorador para criar as pizzas;
  • Utilizar o padrão ponte para separar implementação das interfaces;
  • Utilizar o padrão fachada;
  • Utilizar o padrão peso-mosca;

Utilização dos padrões

O padrão decorator ele tem como proposito adicionar responsabilidades à um objeto dinamicamente. Quando um cliente compra uma pizza, ele tem a opção de pedir uma pizza de mussarela e uma de quatro queijos, de forma a escolher se ela é de cone ou uma tradicional. Para definir os preços das pizzas, esta é realizada dinamicamente, de modo que o preço dela no final é a soma de todos os preços dos ingredientes (cebola, queijos...). Tudo isso dinamicamente.

O padrão Bridge ou ponte tem como proposito desacoplar a abstração da sua implementação. A pizza é criada e adicionada o molho, massa, recheio sem a necessidade de nenhum componente ser modificado. Como pode ver, se eu necessitar criar uma pizza tradicional de mussarela com o dobro de recheio basta apenas eu criar um novo BridgeImp chamado "PizzaTradicionalBridgeImpl" e criar um novo metodo "dobrarRecheio" sem a necessidade de modificar o Builder.

O padrão Facade tem como proposito prover uma interface única para acessar um conjunto de interfaces. Dessa forma, o padrão define uma interface de alto nível para acessar os subsistemas. Para criar uma pizza, é necessário executar uma sequência de atividades que envolvem diversos objetos. Desse modo, o facade da pizzaria é o nosso FabricaPizza. Abaixo segue um exemplo de como ele seria e depois temos o diagrama de classes com ele completo

O padrão Flyweight ou peso mosca tem como objetivo usar o compartilhamento para suportar de forma eficiênte grandes quantidades de objetos. Na nossa pizzaria, as vezes quando tem um rodizio, seria necessário criar diversas pizzas de mussarelas, ou quatro queijos. Desse modo, utilizando o padrão flyweight nós podemos criar as pizzas e compartilha-las de modo a criar menos memoria para o programa.
